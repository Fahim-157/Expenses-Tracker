# Expenses Tracker

A simple and intuitive expense tracking application built with React and Vite. Keep track of your spending with ease, categorize expenses, and view summaries.

## Features

- **Add Expenses**: Easily add new expenses with amount, category, date, and description.
- **View Expenses**: Browse a list of all your expenses in a clean, organized interface.
- **Categorize**: Organize expenses into categories for better tracking.
- **Filter by Category**: Quickly filter expenses to view specific categories.
- **Persistent Storage**: Uses localStorage to save your data locally in the browser.
- **Responsive Design**: Works seamlessly on desktop and mobile devices.
- **Total Summary**: View total number of expenses and total amount spent.

## Technologies Used

- **React 19**: For building the user interface.
- **Vite**: Fast build tool and development server.
- **TailwindCSS**: Utility-first CSS framework for styling.
- **ESLint**: For code linting and quality.

## Installation

1. **Clone the repository**:
   ```bash
   git clone https://github.com/Fahim-157/Expenses-Tracker.git
   ```

2. **Navigate to the project directory**:
   ```bash
   cd Expenses-Tracker
   ```

3. **Install dependencies**:
   ```bash
   npm install
   ```

4. **Start the development server**:
   ```bash
   npm run dev
   ```

5. Open your browser and go to `http://localhost:5173` (or the port shown in the terminal).

## Usage

- **Adding Expenses**: Click on "Add Expense" in the sidebar. Fill in the form with amount, category, date, and description, then submit.
- **Viewing Expenses**: Switch to "Expense List" to see all expenses. Use the category filter to narrow down results.
- **Navigation**: Use the sidebar to switch between adding and listing expenses.
- **Data Persistence**: Your expenses are automatically saved in your browser's local storage.

## Scripts

- `npm run dev`: Start the development server.
- `npm run build`: Build the project for production.
- `npm run lint`: Run ESLint to check code quality.
- `npm run preview`: Preview the production build locally.
